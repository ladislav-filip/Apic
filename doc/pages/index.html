Tři pilíře webových API

Při návrhu RESTových služeb je nutné investovat úsilí do třech klíčových oblastí. První oblastí je dodržení standardů, zejména souvisejících s protokolem HTTP. Druhou mnohem náročnější oblastí je samotné zohlednění REST principů, které obvykle standard HTTP využívají k realizaci avšak nemají jasná univerzální pravidla, kterými by bylo možné se řídit. Poslední oblasti je volba vhodné technologie, která realizaci (technický vývoj) maximálně usnadní.
